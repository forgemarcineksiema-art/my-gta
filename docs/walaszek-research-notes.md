# Walaszek / Blok Ekipa Research Notes

## Scope

These notes translate useful tone patterns from Bartosz Walaszek's animated work into game design rules. They are now reference material only. The approved game canon is original IP: **Blok 13: Rewir**.

This document is not a script bible and does not copy dialogue, characters, episode premises, names, or scenes. It captures production patterns, humor structure, and useful design implications.

## Useful Observations

Walaszek's animated work repeatedly uses low-cost execution as part of the charm. The roughness should feel intentional: simple shapes, ugly poses, blunt timing, and fast punchline delivery.

The humor often comes from an idiotic premise escalating with complete confidence. Characters do not need psychological depth first. They need clear stupidity, strong voice, immediate conflict, and a reason to exist in gameplay.

Blok Ekipa is useful as a reference because it is local and social: a small group, an estate, petty crime, vandalism, alcohol/party chaos, law trouble, and dumb plans that fall apart. That maps naturally to a small sandbox better than to a huge city.

Bogdan Boner/Egzorcysta is useful as a tone reference for short adult animated episodes: absurd threats, dark comedy, rough antiheroes, and brief episode arcs. It suggests missions should escalate quickly and not over-explain.

Kapitan Bomba and other Walaszek work suggest that cheap worldbuilding can still be memorable if names, archetypes, and repeated bits are strong. For Blok 13, that means a few recurring local systems can carry more weight than expensive content.

## Game Translation

### Structure

Use short interactive setups:

```txt
NPC asks for a stupid thing.
Player does a simple action.
World reacts too strongly.
Przypal rises.
Someone comments later.
```

### Characters

Build NPCs as gameplay roles:

- mission starter;
- alarm source;
- gossip source;
- shop/economy blocker;
- chase pressure source;
- vehicle/garage helper;
- future brawl/rival source.

Each role needs:

- a silhouette;
- 3-5 reaction lines;
- a cooldown so they do not spam;
- one mechanical consequence.

### Dialogue

Dialogue should be:

- subtitle-first;
- short;
- punchy;
- aggressive or absurd when appropriate;
- rarely expository;
- tied to gameplay state when possible.

### Visual Style

The 3D style should not chase realism. It should use:

- stylized mid-poly blocks and props with beveled/chamfered edges and modeled detail;
- recessed windows, doors, and shop frontage with real depth;
- exaggerated faces and body shapes later;
- clear readable colors;
- intentionally crude animation;
- dirty estate props with material identity: concrete, dirty plaster, asphalt, metal, glass;
- environmental dirt: stains, cracks, posters, oil marks, chipped curbs;
- strong silhouettes over asset detail;
- target readability: early PS3 / late PS2 era, but cleaner — not photorealistic, not ultra-low-poly.

### Systems Over Volume

The strongest game version is not 50 missions. It is one estate where five systems can collide:

- player movement and comedy collisions;
- Przypal;
- NPC reactions;
- world memory;
- short missions.

## Design Risks

- Copying episodes too literally would make the game feel like a checklist, not a sandbox.
- Too much random slapstick would make controls feel broken.
- Too many NPC lines without consequences would feel like noise.
- Expanding the map before the estate reacts would dilute the best idea.
- Over-polishing the visuals could weaken the dirty, cheap comedy identity.
